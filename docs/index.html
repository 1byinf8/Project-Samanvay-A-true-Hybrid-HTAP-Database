Each component is designed for maximum performance and reliability

Lock-Free Skiplist Memtable

The memtable is the in-memory write buffer absorbing all incoming writes before persistence. Our lock-free implementation enables high concurrency without mutex contention.

‚úì Compare-and-swap based inserts
‚úì Logical deletion with tombstones
‚úì O(log n) expected operations
‚úì Range query via lowerBound() 
class Skiplist {
private:
    static const int MAX_LEVEL = 16;
    std::atomic<int> currentMaxLevel;
    Node* header;
    
public:
    // CAS-based concurrent insert
    bool add(const Entry& key);
    
    // Lock-free search
    bool search(const Entry& target);
    
    // Logical deletion
    bool erase(const Entry& key);
    
    // Range query support
    Iterator lowerBound(const string& key);
};

LSM-Tree Level Manager

Organizes persistent storage into multiple levels with automatic row-to-columnar conversion. Write amplification is minimized through size-tiered compaction.

Level Format Size Limit
L0 Row (SSTable) 64 MB
L1-L3 Row (SSTable) 640 MB - 64 GB
L4+ Columnar Unbounded 
class LSMTreeManager {
public:
    // Add SSTable to level
    void addSSTable(const SSTableMeta& meta);
    
    // Query support
    vector<SSTableMeta> getSSTablesForKey(
        const string& key);
    
    // Compaction control
    bool shouldTriggerCompaction(int level);
    
    // Check if level uses columnar
    bool isColumnarLevel(int level) const {
        return level >= 4;
    }
};

Write-Ahead Log (WAL)

Ensures durability by persisting all operations before they're applied to the memtable. Binary format with fsync guarantees data survives crashes.

struct WALEntry {
    uint64_t sequenceNumber;
    uint64_t timestamp;
    Operation operation;  // INSERT or DELETE
    string key;
    string value;
};

class WAL {
public:
    bool append(uint64_t seq, Operation op,
               const string& key, const string& val);
    void sync();  // fsync to disk
    vector<WALEntry> recover();
    void truncate(uint64_t upToSeq);
};

SSTable (Sorted String Table)

Immutable on-disk structures with Bloom filters for O(1) negative lookups and sparse indexes for O(log n) seeks.

Header Version, sequences, key range
Data Section Sorted key-value entries
Sparse Index Every Nth key ‚Üí offset
Bloom Filter ~1% false positive rate 
class SSTable {
public:
    // Create from sorted entries
    bool create(const vector<Entry>& sorted);
    
    // Point lookup with Bloom filter
    optional<Entry> get(const string& key);
    
    // Range scan
    vector<Entry> rangeQuery(
        const string& start,
        const string& end);
    
    // Load from disk
    bool load();
};

Intelligently routes queries between row-store (OLTP) and columnar (OLAP) paths based on query characteristics and data locality.

Full Scan ‚Üí Both Paths

Quick Start Guide

Build and run Samanvay in under 5 minutes

1

Clone the Repository

git clone https://github.com/your-repo/Project-Samanvay.git
cd Project-Samanvay
2

Build with CMake

# Create build directory
mkdir build && cd build

# Configure and build
cmake ..
cmake --build .
3

Run Tests

# Quick validation
./test_skiplist && ./test_htap

# Stress test (1M records, 8 threads)
./stress_test 1000000 8
4

Use in Your Project

#include "storage_engine.hpp"

int main() {
    storage::StorageEngine engine;
    
    // OLTP: Point operations
    engine.put("user:1", "Alice");
    auto val = engine.get("user:1");
    
    // OLAP: Aggregation query
    auto req = QueryRequest::aggregation(
        AggregationType::SUM, "amount");
    auto result = engine.executeAggregation(req);
}
